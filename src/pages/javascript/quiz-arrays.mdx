Here is a bunch of questions related to JavaScript arrays. It helps a developer to test his/her knowledge about JavaScript arrays.

**Question 1:**

Which of the following is / are valid statement(s) to declare an array?

A. `const arr = [2, 4, 6];`

B. `const arr = new Array(2, 4, 6);`

C. `const arr = Array(2, 4, 6);`

</Question>

<Option>

A and B

</Option>
<Option>

Only A

</Option>
<Option>

B and C

</Option>
<Option isAnswer="yes">

A, B and C

</Option>

<Explanation>

**Explanation:** <br/>
Declaring an array using square brackets(`[]`) is common. Talking about `Array` function in JavaScript, it can be used both as a constructor and a normal function. That means, `Array()` is an overloaded function in JavaScript. We can use both the techniques to declare an array.

</Explanation>

<Question>

**Question 4:**

What is the output?

```javascript
const arr = Array(3);
console.log(arr);
```

</Question>

<Option>

`undefined`

</Option>
<Option>

`[3]`

</Option>
<Option isAnswer="yes">

`[undefined, undefined, undefined]`

</Option>
<Option>

None of these

</Option>

<Explanation>

**Explanation:** <br/>
`Array()` function in JavaScript can be used to declare arrays. When more than one argument is passed to `Array()` function it creates an array with all the passed arguments as array elements.

```javascript
const arr = Array(3, 6, 9);
console.log(arr); // [3, 6, 9]
```

If a **single numeric argument** is passed to `Array()`, it takes the argument as the length of the array.

```javascript
const arr = Array(3);
console.log(arr); // [undefined, undefined, undefined]
```

If a single non-numeric argument is passed to `Array()`, it is taken as the array element.

```javascript
const arr = Array("3");
console.log(arr); // ["3"]
```

> When we use `Array.of()`, even if we pass a single numeric argument, that number forms the array element. It is not taken as the length of array.

```javascript
console.log(Array(3)); //[undefined, undefined, undefined]
console.log(Array.of(3)); // [3]
```

</Explanation>

<Question>

**Question 8:**

In what all ways, we can check if a variable `arr` contains an array in JavaScript?

A. `Array.isArray(arr) === true`

B. `arr.constructor.name === "Array"`

C. `typeof arr === "Array"`

</Question>

<Option isAnswer="yes">

A and B

</Option>
<Option>

Only A

</Option>
<Option>

B and C

</Option>
<Option>

A, B and C

</Option>

<Explanation>

**Explanation:** <br/>
`isArray()` is a static method of `Array()` function which can be used to check if a value is an array or not.

Option _B_ also works. Every array is an instance of `Array()` function. So the `constructor.name` always point to the name of `Array()` function, ie `"Array"`.

Option _C_ does not work. It is because `typeof arr` returns `"Object"` instead of `"Array"`. So we cannot use `typeof` operator.

</Explanation>
</QuestionItem>

<Question>

**Question 10:**

```javascript
const arr1 = ["Apple", "Banana"];
const arr2 = arr1.concat();
arr2[0] = "Pineapple";
console.log(arr1);
```

</Question>

<Option isAnswer="yes">

`["Apple", "Banana"]`

</Option>
<Option>

`["Pineapple", "Banana"]`

</Option>
<Option>

`SyntaxError: missing argument for concat()`

</Option>
<Option>

`["Apple", "Banana", "Pineapple"]`

</Option>

<Explanation>

**Explanation:** <br/>
`Array.prototype.concat()` methods returns a **new array**. Also, it is not mandatory to pass an argument to `concat()` method. If we are not passing an argument, `concat()` method returns a shallow copy of `arr1` and assign to `arr2`. That is why, when the element in `arr2` is changed, `arr1` is not affected.

</Explanation>
</QuestionItem>
